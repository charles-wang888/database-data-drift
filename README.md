# 实验场景说明

这是基于 LLM 识别数据库表中的数据漂移点的演示例子。场景如下：

假设有一张数据库表（比如交易表），里面记载了不同人的购买不同商品的行为。突然有一天，发现交易表中的数据发生了漂移，体现出不同的特征，比如某一天购买数量明显减少，或者某个人的购买行为在一段时间内明显激增等。但是这些特征通过肉眼很难挖掘出来。数据运营人员也并不清楚数据库中的数据是否发生了数据漂移，发生了什么样的数据漂移，发生数据漂移的偏离系数如何，如何归因（即找到发生对应数据漂移的根因）。

本 Demo 主要演示了以下内容：

1. 如何基于 tsfresh 框架从数据库表中提取特征因子。
2. 如何利用 LLM 来分析特定的特征因子是否发生了数据漂移，并关联到漂移点（即发生数据漂移的具体时间）。
3. 归因。基于数据库表的数据，找到数据漂移点的根本原因。

---

# 实验过程

整个实验过程分为四个步骤：

## 1. 数据准备

这里假设有一张交易表，表结构为：  
`id`, `user_id`（用户id）, `product`(商品), `price`(价格), `quantity`(购买数量), `trade_time`(交易时间)。  
商品只有五种类型，分别是：汽水、矿泉水、果汁、茶饮、咖啡。  
为了模拟真实情况，这里的矿泉水、汽水、咖啡的 price 都设置了价格区间，让其在特定的价格区间内随机赋值，使其更能模拟真实情况，比如汽水会比矿泉水贵，咖啡更贵。

## 2. 利用 tsfresh 识别交易数据特征因子

简单描述代码的相关逻辑：

- 以天为单位（即数据库表中的 trade_date 交易日期列）来聚合平均价格和总购买数量。
- 利用 tsfresh 的 `extract_features` 方法来基于聚合数据提取特征因子，并按照交易时间（trade_date）排序，把提取到的交易数据特征因子放入 `drift_feature.csv` 文件中。
- 把日期 id 与真实日期之间的映射关系存入 `id_data_map.csv` 中，这是因为在 tsfresh 中只会包含日期 id，但是在对外输出漂移点发生日期并结合交易行为归因时，需要精确展示具体日期，因此要有这样的日期映射表。

## 3. 提取最重要的特征因子

不是所有的特征因子都对数据漂移识别和归因分析有很大的帮助，因此调用 `filter_drift_feature.py` 来过滤并提取最重要的特征因子。

怎样算最重要的特征因子呢？一般来说要结合业务视角来表征的，会涉及到复杂数据建模分析。这里 Demo 简化起见，从纯技术角度，利用“方差筛选”的方式，来识别最重要的特征因子。

## 4. 利用 LLM 分析漂移点、漂移系数并归因分析

接下来的工作重点，就是利用 LLM 来分析漂移点（何时发生了数据漂移）以及漂移系数（漂移力度）并做归因分析了（因为哪些原因导致了数据漂移）。

# 关键技术细节

1. 分析数据漂移是个非常强的“推理类”场景，而不是“事实作答”场景，因此需要找一个推理能力比较强的 LLM 来胜任此事，这里选择了 qwen3:4b。qwen3 可以开启推理模式，从而会有 `<think>` 输出。也可以用 deepseek-r1。
2. 数据异常点的检测方式是 z-score，它是一种数据统计方法，用来描述一个值与一组值的平均值之间的偏离程度的统计测量分数，这个与数据漂移的本质定义是一致的。
3. 为了找到数据漂移点，不仅要给出交易数据，也要给出前面提取到的关键 top-10 个特征因子，所以要同时把这两者传给 LLM。
4. 特征因子很多，即便是 top-10 也有 10 个，所以我们一般指定针对某个特征因子进行数据漂移识别和归因分析。因此代码中用 `FEATURE_TO_ANALYZE = <要分析的特征名>` 来指定寻找针对某个具体特征因子的数据漂移。如果要分析其他的特征因子的数据漂移，只要结合上个步骤 `filter_drift_feature.py` 的输出，把 `<要分析的特征名>` 替换为其他特征因子名称即可。
5. 对于整个数据库表数据而言，分析所有的数据漂移点需要耗费大量的 token，所以可以只分析局部的数据。因此用 `TOP_N_DRIFT_POINTS = <要分析的前N个漂移点数量>` 来指定要分析几个数据漂移点。当然也可以分析全部，这会导致执行时消耗非常长的时间和巨大的 token 数量。
6. 找到了漂移点之后，需要结合漂移点（时间）附近的交易数据进行归因，归因也是 LLM 完成的，它来推断发生上述数据漂移的根本原因是什么。归因难度很高，`<think>` 过程会消耗海量的 token，因此我在 Prompt 中故意加了以下两句话（标蓝），否则最终的输出篇幅可能几百页，已尝试过：

   - **注意：特征因子的计算方法已经由专业算法（tsfresh）完成，无需你推理或反思特征的计算方式。**
   - **请只关注给定的特征值、日期和交易明细，进行归因和解释，不要讨论特征的计算过程，思考过程尽量精简。**
